## Description

This commit adds comprehensive documentation for the NEOPRO fleet management system, making it easy for administrators to manage the entire fleet of Raspberry Pi devices.
## Documentation added
### ADMIN_GUIDE.md (Complete administration guide)
- **Accessing the dashboard**: Login, roles (admin/operator/viewer)
- **Adding a device to the fleet**:
  - Step 1: Register site in dashboard
  - Step 2: Install agent on Raspberry Pi
  - Step 3: Verify connection
- **Managing sites**:
  - View, filter, search sites
  - Site details with real-time metrics
  - Actions: restart service, view logs, reboot, API key management
  - Edit and delete sites
- **Managing groups**:
  - Create groups (sport, geography, version, custom)
  - Add/remove sites from groups
  - Group actions: deploy content/updates, restart services/systems
- **Deploying content**:
  - Upload videos
  - Deploy to individual sites or groups
  - Track deployment progress in real-time
- **Deploying updates**:
  - Create software versions with release notes
  - Critical update marking
  - Deployment options (auto-rollback, auto-reboot)
  - Version distribution tracking
- **Monitoring the fleet**:
  - Dashboard statistics
  - Real-time notifications
  - Site status indicators
  - Automatic alerts (temperature, CPU, RAM, disk)
- **Troubleshooting**:
  - Site not showing as online
  - Deployment stuck at 0%
  - Metrics not updating
  - Invalid API key errors
  - Post-update errors
  - Performance issues (high CPU/RAM)
  - Disk space saturation
- **Best practices**: Fleet management, security, updates, maintenance
### QUICK_START.md (Quick start guide)
- **3-step process** to add first device (7 minutes total):
  1. Create site in dashboard (2 min)
  2. Install agent on Raspberry Pi (5 min)
  3. Verify connection (30 sec)
- **Next steps**: Organize sites, deploy content, deploy updates
- **Useful commands** for Raspberry Pi management
- **Troubleshooting** quick fixes
